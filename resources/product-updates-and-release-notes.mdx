## Changelog Format

Each release includes:

**Version Number**
Qumis uses semantic versioning (e.g., 2.1.0)

**Release Date**
The date the update became available to all users

**New Features**
Completely new capabilities available in this release

**Improvements**
Enhancements to existing features, performance optimizations, and user experience improvements

**Bug Fixes**
Issues resolved in this release

**Breaking Changes** (if applicable)
Any changes that may affect your workflows or integrations

**Deprecations** (if applicable)
Features being retired or replaced in future versions

## Staying Informed
